# Deploy Guide — YouTube transcript service

The Next.js app calls a small Python service (`transcript-service-py/`) to fetch YouTube subtitles. Deploy that service to Render, then point Vercel at it.

## Phase 2 — Deploy the Python service to Render

1. Push the repo to GitHub (Render pulls from there).
2. Render dashboard → **New → Blueprint** → pick this repo → it auto-detects `transcript-service-py/render.yaml`.
   - (Or: **New → Web Service**, runtime **Docker**, root dir `transcript-service-py`.)
3. Set env vars on the new service:
   - `API_SECRET` — auto-generated by the blueprint. **Copy the value** — you'll paste it into Vercel.
   - `YOUTUBE_COOKIES_B64` — *optional but recommended.* Reduces YouTube bot-detection blocks.
     - Export your YouTube cookies (Netscape format) using a browser extension like **"Get cookies.txt LOCALLY"**.
     - Encode it:
       - PowerShell: `[Convert]::ToBase64String([IO.File]::ReadAllBytes("cookies.txt"))`
       - Linux/macOS: `base64 -w0 cookies.txt`
     - Paste the resulting single-line string as the value.
   - `PROXY_URL` — *optional.* Only set if Render's IPs get blocked (format: `http://user:pass@host:port`).
4. Deploy. Once live, hit `https://<your-service>.onrender.com/health` — should return `{"status":"ok",...}`.
5. Copy the service URL — you'll need it next.

> **Note (free tier):** Sleeps after 15 min idle, ~30s cold start on first request after sleep. Upgrade to Starter ($7/mo) if that's a problem.

### Keep the free service warm (recommended)

So users never hit the 30s cold start, ping `/health` every ~10 min from any external uptime monitor. Vercel Hobby cron only runs ~2/day, so use a third-party pinger:

- **UptimeRobot** (free, easiest): New Monitor → HTTP(s) → URL `https://<your-service>.onrender.com/health` → interval 5 min.
- **cron-job.org** (free): same idea — create a job hitting `/health` every 10 min.

This is purely external; nothing in the Next.js or Python code depends on it.

## Phase 3 — Wire up Vercel

In your Vercel project → **Settings → Environment Variables**, add:

| Key | Value |
|---|---|
| `TRANSCRIPT_API_URL` | The Render service URL (e.g. `https://unbiasy-transcripts.onrender.com`) |
| `TRANSCRIPT_API_SECRET` | The same value as `API_SECRET` from step 2.3 |

Redeploy. Done.

## Verify

- Open the deployed app → paste a YouTube URL with captions → confirm analysis runs.
- Try a video with captions disabled → confirm you see *"This video doesn't have captions available"* (not a yt-dlp error).
